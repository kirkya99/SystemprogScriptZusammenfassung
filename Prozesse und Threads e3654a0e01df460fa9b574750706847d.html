Prozesse und Threads

Prozesse in Linux

  • Bei Starten eines Programms wird für dieses Programm ein Prozess im BS erzeugt
    • Jeder Prozess erhält vom BS eine Kennung die PID
  • Prozesse stehen in Vater-Kind-Beziehung
    • Ein Prozess kann Kindprozesse erzeugen und auf deren Beendigung warten
  • Jeder Prozess hat seinen eigenen geschützten Adressraum, in dem das Programm (Codebereich im Speicher) und die Daten (Datenbereich, Stack, Heap) liegen
    • Ein Prozess kann nicht auf Daten im Adressraum eines anderen Prozesses zugreifen
    • Wollen zwei Prozesse auf gemeinsamen Daten arbeiten, geht das nur über das Betriebssystem ( →\to→﻿ shared Memory)

Prozess erzeugen

  • Erzeugen eines Prozesses in Linux erfolgt in 2 Schritten

    1. Schritt: Kindprozess erzeugen →\to→﻿ Systemfunktion fork()

    2. Schritt: Neues Programm in den Kindprozess laden →\to→﻿ Systemfunktion exec()

  • Systemfunktion fork()
    • Erzeugt eine Kopiedes aktuellen Prozesses. Kopie wird zu Kindprozess

      Der Kindprozess übernimmt die Daten, den Programmcode, den Befehlszähler und die Dateideskriptoren vom Elterprozess

    • Nach erfolgreichem fork-Aufruf gibt es 2 Prozesse mit identischen Daten und Programm. Untscheidung in welchem Prozess man sich befindet nur über return-Wert von fork möglich
  • in der Regel soll im Kindprozess ein eigenes Programm abgearbeitet werden →\to→﻿ man muss im Kindprozess den Programm- und Datenbereich mit einem neuen Programm überladen

→\to→﻿ Systemfunktion int execl(const char *path, const char *arg, ...);

Prozesse in Linux

Warten auf Ende einse Prozesses

  • Wenn ein Prozess sich beendet, übergibt der Prozess dem Betriebssystem einen Status. Dieser Status definiert, ob der Prozess mit oder ohne Fehler beendet wurde.
  • Solange der Status im Betriebssystem gespeichert wird, ist der Prozess zwar tot, aber doch noch nicht richtig →\to→﻿ Prouess hat den Zustand Zombie
  • Um den Prozess endgültig zu töten muss der Status vom Vaterprozess vom Betriebssystem abgeholt werden
  • Beim Abholen des Status kann der Vaterprozess entscheiden, ob er auf das Ende des Prozesses warten möchte oder ob er solange nachfragt, bis der Prozess beendet wurde (Polling).
    • pid_t wait(int status); Wartet auf das Ende eines Prozesses. In status wird der Status des beendeten Prozesses geschrieben. Return-Wert ist die pid des beendeten Prozesses
    • pid_t waitpid(pid_t pid, int *wstatus, int options); Holt sich den Status des Prozesses mit der angegebenen pid. Der Vater kann auf das Ende des Prozesses warten oder nicht.

Threads in Linux

  • Innerhalb eines Prozesses können Funktionen nebenläufig in Threads ausgeführt werden
  • Threads stehen nicht in Vater-Kind-Beziehung
    • Zu jedem Prozess wird mindestens ein Thread erzeugt. Das ist der Haupthread, in dem main abgearbeitet wird
  • Jeder Thread hat seinen eigenen Stack
  • Alle Threads teilen sich den Datenbereich im Speicher und alle geöffneten Dateien
  • Wann benötigt mann mehrere Threads?
    • Client-Server-Anwendung
      • Server muss mehrere Clients gleichzeitig bedienen. Für jeden Client wird ein Thread erzeugt. Jeder Thread kann dann nebenläufig arbeiten und mit seinem Client kommunizieren
    • Prozess, der gleichzeitig auf mehrere Eingaben reagieren muss z.B. EIngaben von der Konsole und Nachrichten, die über das Netzwerk kommen
    • Prozess, der fortlaufend Ausgaben produziert aber auch gleichtzeitig auf Eingaben reagieren muss

    Threads erzeugen

    • Für jeden Thread benötigt man eine Funktion
    • Im Haupt-Thread wird die main-Funktion abgearbeitet
    • Für jeden weiteren Thread muss man eine FUnkion mit bestimmbter Signatur deklarieren
      • Signatur Thread-Funktion: void * start_routine (void *)
    • System-Funktion: pthread_create() erzeugt einen Thread
    #include <pthread.h>
int pthread_create(pthread_t *thread, const pthread_attr_t * attr, 
											void * (* start_routine)(void *), void * arg);

  • Thread beenden
    • freiwillig: void pthread_exit(void *retval);
    • unfreiwillig: int pthread_cancel(pthread_t thread);
    • Auf Thread-Ende warten: int pthread_join(pthread_t thread, void **retval);
