Zeiger in C

Grundlagen

Zeiger allgemein

Der Adress-Operator &

  • Jede Variable belegt eines oder mehrere Bytes im Speicher
  • Position des ersten Bytes im Speicher ( = Adresse) kann über Adress-Operator (&) abgefragt werden
  • Ergebnis ist ein typisierter Zeiger auf die Variable
  • In diesem Beispiel: Zeiger auf char
  • Schreibweise in C: 
    char *

Zeigervariablen

Zeiger lassen sich in sogenannten Zeigervariablen speichern, z.B.:

double sloppy_pi = 3.14;
double *pi_ptr = &sloppy_pi;

int answer = 42;
int *answer_ptr ) &answer;

char *dangerous //you will see why ...

Der Dereferenzierungs-Operator *

Auf den SPeicher, auf den ein Zeiger verweist, kann über den Dereferenzierungs-Operator (*) zugegriffen werden:

int number = 123;
int *ptr = &number;

printf(%i\n", *ptr); //will print 123

*ptr = 456;

printf("%i\n", *ptr); //will print 456
printf("%i\n", number); //will also print 456

Vorsicht, wilde Zeiger!

int *born_to_be_wild;

printf("%i\n", *born_to_be_wild); //BOOM!
*born_to_be_wild = 1; //BOOM!

Nur lesende Zeiger

int the_answer = 42

const int *ptr = &the_answer;
printf("%i\n", *ptr); //ok
*ptr = 21; //compiler error!
const int the_answer = 42

const int *ptr = &the_answer; //ok
*ptr2 = &the_answer; //compiler warning!

Zeiger auf Zeiger

Auch von Zeigervariablen kann man sich die Adresse besorgen. Das Ergebnis ist ein Zeiger auf einen Zeiger:

int number = 123;
int *ptr = &number;
int **ptrptr = &ptr;

printf("%i\n", **ptrptr); //prints 123

//If you're really crazy:
int ***ptrptrptr = &ptrptr;

Nullzeiger und untypisierte Zeiger

Der Nullzeiger

Hat ein Zeiger den Wert NULL, dann verweist dieser bewusst auf keinen gültigen Speicherbereich. Im Gegensatz zu einem wilden Zeiger, lässt sich überprüfen, z.B.:

if(ptr == NULL)
{
	//DO NOT DEREFERENCE; NOT BY ANY MEANS!
}
else
{
	//might be dereferenced here
}

Untypisierter Zeiger

int number = 123;
char ch = 'x';

void *ptr = &number;

ptr = &ch;
char *chptr = ptr;
  • Zu lesen als: Zeiger auf irgendetwas
  • Untypisierter oder generischer Zeiger genannt
  • Keine Überprüfung durch den Compiler, ob der Zeigertyp passt
  • Eingesetzt bei generischen Funktionen, die unabhängig vom Datentyp arbeiten

Zeiger und Arrays

Zugriff auf Arrays über Zeiger

Interne Darstellung von Arrays

Intern ist ein Array nur ein Block von mehreren Elementen der gleichen Größe:

int numbers [] = { 1, 4, 9, 16, 25 };

Name des Arrays als Zeiger

Der Name eines Arrays ist ein Zeiger auf das erste Element des Arrays.

int numbers [] = { 1, 4, 9, 16, 25 };

printf("%p\n"), numbers);
printf("%p\n"), &numbers[0]); //same as above, just clumsier

Pointer-Arithmetik

Man kann mit Zeigern rechnen:

int numbers [] = { 1, 4, 9, 16, 25 };

printf("%i\n"), numbers[2]);

//or, use pointer arithmetics:
printf("%i\n"), *(numbers+2));

Zeiger-Arrays

Zeiger-Arrays allgemein

Man kann auch Arrays definieren, die Zeiger enthalten:

const char *greetings [] = { "Hallo", "Servus", "Moin", "Hi" };
const size_t num_greetings = 
	sizeoff(greetings) / sizeof(*greetings);

printf("%s!\n", greetings[rand() % num_greetings]);

Anwendungsbeispiel: Auswertung der Kommandozeile

int main(int argc, char *argv [])

Zeiger und Strukturen

Zeiger als Elemente von Strukturen

Natürlich können Strukturen auch Zeiger als Elemente enthalten:

struct Node
{
	char name[32];
	void *data;
	struct Node *next;
};

Zeiger auf Strukturen und Strukturelemente

Zeiger auf Strukturen und Strukturelemente

struct __attribute__((__packed__))
MyStruct
{
	uint32_t u32;
	uint16_t u16;
};

struct MyStruct s;
struct MyStruct *sp =&s;
uint32_t *u32p = &s.u32;
uint16_t *u16p = &s.u16;

Der Pfeil-Operator ->

Hat man einen Zeiger auf eine Struktur, dann kann man über den Pfeil-Operator auf die Komponenten der Struktur zugreifen, auf die der Zeiger verweist:

struct Something
{
	int num;
	double *dp;
};

double d= -1.0;
struct Something s;
struct Something *ptr = &s
(*ptr).num = 42; //clumsy
ptr->num = 42;

ptr->dp = &d
*ptr->dp = 3.14

Zeiger und Funktionen

Zeiger als Rückgabewerte und Parameter von Funktionen

Zeiger als Rückgabewerte von Funktionen

Funktionen könne grundsätzlich auch Zeiger zurückgeben.

Arrays werden bei der Rückgabe nicht kopiert, sondern zu Zeigern. Deshalb Vorsicht:

  • Falsch: Zeiger auf lokale Variable zurückgeben
    char *func(void)
{
	char result [32];
	//...
	return result;
}
  • Möglichkeit: Speicher dynamisch reservieren
    char *func(void)
{
	char *result = malloc(32);
	if(result == NULL)
		return NULL;
	//...
	return result;
}

Übergabe von Zeigern an Funktionen

Funktionen können Zeiger als Parameter entgegen nehmen:

void change_value(int *x)
{
	*x = 42;
}

Warum braucht man das?

  • Parameter werden kopiert
    • Primitive Datentypen (int, double, char, usw…) werden bei der Übergabe kopiert
      • Soll die Funktion den Wert ändern, muss ein Zeiger auf die entsprechende Variable übergeben werden
      • “Call by reference”
    • Auch Strukturen und Unions werden kopiert
      • Aus Performance-Gründen große Strukturen imm erper Zeiger übergeben
      • const verwenden, wenn Struktur nicht geändert werden soll
  • Arrays werden zu Zeigern
    • Arrays werden nicht kopiert, sondern als Zeiger auf das erste Element übergeben
      • Array-Übergabe funktioniert nur muit Zeigern!
        • Gilt insbesondere auch für C-Strings!
        • Größenangabe wird nicht mitgeführt, kein array.length wie in Java!

Definition von Zeiger-Parametern

Folgende Funktionsprototypen bedeuten für den Compiler exakt das gleiche:

void f(char *array);
void f(char array[]);
void f(char array[60000000]);

Beispiel: Übergabe großer Strukturen

struct Person
{
	char first_name [32];
	char last_name [32];
};

void print_person(const struct Person *p)
{
	printff("%s %s\n", p->first_name, p->last_name);
}

Beispiel: Ergebnis der Funktion ist ein String

void func(char *result, size_t result_size)
{
	//...
}

Oft einfacher und schneller, als die zuvor gezeigte Variante mit dynamischer Speicherverwaltung!

Zeiger auf Funktionen

Zeiger auf Funktionen allgemein

  • Auch Funktionen haben eine Startadresse, die abgefragt werden kann
    void print_hello(void)
{
	puts("Hello, world!");
}

int main(void)
{
	void (*func_ptr)(void) =
		print_hello;
	func_ptr();
}
  • Funktionsname (ohne Klammern) liefert diese
  • Ergebnis ist ein Funktionszeiger
  • Funktion kann auch über Zeiger aufgerufen werden

Beispiel: Generisches Iterieren über eine verkettete Liste

typedef struct Node
{
	char text [128];
	struct Node *next;
};

void for_each_node(Node *start, void (*what)(Node *))
{
	Node *next;
	for(Node *it=start; it!=NULL; it=next)
	{
	next = it->next;
	what(it);
	}
}

Verwendung z.B..:

void print_node(Node *n)
{
	puts(n->text);
}

//...

for_each_node(start, print_node);

oder:

void free_node(Node *n)
{
	free(n);
}

//...

for_each_node(start, free_node);

Systemfunktionen mit Zeigern

Beispiele für Systemfunktionen mit Zeigern

Zahlreiche Systemfunktionen arbeiten mit Zeigern. Eine kleine Auswahl:

  • Einlesen von Daten aus einer Datei: 
    	ssize_t read(int fd, void *buf, size_t count);
  • Erzeugen von Threads:
    int pthread_create(pthread_t *thread, const pthread_attr_t *attr,
void *(*func)(void *), void *arg);
  • Einbinden von Dateien oder Shared Memory in den Adressbaum;:
     void *mmap(void *attr, size_t length, int prot, int flags, int fd, off_t offset);
