Strukturen

  • Datenobjekte mit mehreren Komponenten
typedef struct myStruct
{ char name [32];
	unsigned int age;
	float size; } myStruct;
  • Jede Komponente kann einen eigenen Datentype haben
  • Datentyp der Komponente kann auch ein Array der wieder eine Struktur sein →→→﻿ Verschachtelung von Strukturen ist möglich
  • Wo wird eine Struktur benötigt?
    • Speichern von Datensätzen aus einer Datenbank
    • Versenden/Empfangen von Nachrichten über das Netzwerk an ein anderes Programm (z.B. einen Server)
#define TEXT 1
#define BINARY 2
#define MIXED 3

typedef struct
{ char name[32];
	char type;
 	unsigned int lengthBody;
} messageHeader;

typedef struct
{ char number;
	int zahl;
	char title[6];}
} messageBody;

typedef struct
{ messageHeader header;
  messageBody body;
} message;
  • Speicherung von Strukturen durch den Compiler
    • Der Compiler speichert die Komponenten einer Struktur nicht immer direkt hintereinander im Speicher, sondern kann Lücken im Speicher lassen, um die Zugriffsgeschwindigkeit zu optimieren
    typedef struct
{ char number;
	int zahl;
	char title [6];
} messageBody;
    • Erwartete Größe →→→﻿ 11
    • Tatsächliche Größe →→→﻿ 16 (4+4+8)
  • Packen von Strukturen
    • Man kann den Compiler anweisen bei Strukturen die Komponenten im Speicher direkt aneinander zu reihen
      • Mit der Direktiven #pragma pack(1) oder
      • über ein Compilerattribut __attribute__((__packed__)) 

    typedef struct __attribute__((__packed__))
{ char number;
	int zahl;
	char title [6];
} messageBody;
    • Erwartete Größe →→→﻿ 11
    • Tatsächliche Größe →→→﻿ 11

Unions

  • Unions
    • Unions sind ähnlich einer Struktur, mit dem Unterschied, dass die Komponenten der Union im Speicher überlagert werden und nicht wie bei einer Struktur nebeneinander liegen
      • Die Union verbraucht im Speicher nur so viel Platz wie die größte Komponente benötigt

        Achtung: Von einer Untion kann man zu einem Zeitpunkt nur genau eine Komponente verwenden

      • Eine Struktur braucht Speicherplatz für alle Komponenten
        • Man kann auf alle Komponenten gleichzeitig zugreifen
    typedef struct
{ char number;
	int zahl;
	char title[6];
} messageBody

    Anordnung im Speicher

    typedef union
{ char number;
	int zahl;
	char title[6];
} messageBody;
  • Wann macht es Sinn eine Union zu verwenden?
    • Bei der Netzwerkprogrammierung
      • Will man über das Netzwerk nacheinander unterschiedliche Nachrichten versenden oder empfangen, bracht man für jeden Nachrichtentyp eine andere Struktur
      • Da man zu einer Zeit nur eine Nachricht bestimmten Typs verarbeitet, kann man sich einen Union vauen, deren Komponenten die Strukturen der Nachrichtentypen enthält.
  • Nachrichten
  • Im Kopf der Nachricht git man an, welchen Nachrichtentyp man verwendet
  • Der Inhalt der Nachricht kann sein:
    • reine Textdaten
    • reine Binärdaten
    • Mischform

Integer-Datentypen

  • Die Sprache C macht bei ganzzahligen Datentypen wie int, unsigned, short und long keine Angabe wieviel Speicher diese tatsächlich auf der Rechnerhardware belegen. Damit sind diese Datentypen abhängig von der Rechnerarchitektur.
  • In speziellen Fällen (z.B. Netzwerkprogrammierung) möchte man aber die Länge einer Variable im Speicher fest vorgeben

    Dazu werden in stdint.h die folgenden Datentypen deklariert:

    int8_t int8_t int16_t int32_t int64_t uint8_t uint8_t uint16_t uint32_t uint64_t
